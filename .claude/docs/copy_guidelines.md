# COPY_GUIDELINES.md — Anti-Cliche Lyric Rules

**For:** Builder AI
**Project:** NomadMeet
**Established:** 2026-02-26
**Purpose:** All user-facing copy in the app must pass these rules before shipping.

---

## The Core Test

> "Would a real person actually phrase it like this?"

If the answer is no, rewrite it.

---

## Rule 1: Human > Polished

Keep it human. Slightly messy is better than polished. Specific is better than cosmic.

- **Yes:** "Nothing nearby yet. Drop the first pin."
- **No:** "Discover amazing events happening around you!"

---

## Rule 2: Banned Words

Never use these in any user-facing string:

| Banned          | Why                                        |
|-----------------|--------------------------------------------|
| Epic            | Overused, means nothing                    |
| Neon            | Visual cliche, not a real descriptor       |
| Shadows         | Vague aesthetic filler                     |
| Digital Ghosts  | Pretentious, no one talks like this        |
| Journey         | Every app says "journey"                   |
| Discover        | Corporate travel brochure energy           |
| Unlock          | Gamification cliche                        |
| Elevate         | LinkedIn energy                            |
| Seamless        | Marketing filler, never means anything     |
| Curated         | Pretentious for a backpacker app           |

---

## Rule 3: CTA Buttons

CTAs should sound like something you'd say out loud to a friend.

| Old (Cliche)         | New (Human)        |
|----------------------|--------------------|
| Join Event           | Count me in.       |
| Join this Journey    | Count me in.       |
| Discover Events      | Nearby             |
| Open Chat  →         | Open chat →        |
| Join Meetup          | Count me in.       |
| Discovery Area       | Nearby             |
| Filter Events        | Filter             |

---

## Rule 4: Empty States

Empty states are where the app's personality shows. Write them like a friend texting, not a product manager writing a spec.

| Old                                         | New                                        |
|---------------------------------------------|--------------------------------------------|
| No events nearby. Drop a pin to start one!  | Nothing nearby yet. Drop the first pin.    |
| This event has ended — chat is read-only.   | This one's over — chat is read-only.       |
| No meetup point yet                         | No meetup point yet *(this one's fine)*    |

---

## Rule 5: Error Messages

Errors should be honest, not corporate.

- **Yes:** "Couldn't save that. Try again?"
- **No:** "An unexpected error has occurred. Please contact support."

---

## Rule 6: The Sniff Test

Before any copy ships, read it out loud. If it sounds like:
- An airline safety card → rewrite
- A startup pitch deck → rewrite
- A friend at a hostel bar → ship it

---
*End of Copy Guidelines*
